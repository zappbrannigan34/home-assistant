# CO2 Ventilation Control — Drivent V2 (zap_left)

Автоматическое управление проветриванием через оконный привод Drivent V2 на основе уровня CO2.

---

## Почему не PID?

PID-регулятор дёргает привод на **каждое измерение**: CO2 постоянно меняется из-за внешних условий, и система никогда не приходит в равновесие. Результат — привод работает каждую минуту, изнашивается за год.

Этот пакет использует **зонное управление по тренду** (аналог Drivent Air):
- Смотрим **направление тренда** CO2 (растёт / падает / стабильно)
- Двигаем привод **только когда тренд подтверждает необходимость**
- Шагами по **5% или 2%** (не микродвижения по 1%)
- Оценка каждые **5 минут** (не каждую минуту)

Реальных движений привода: **20-40 в день** вместо 1440 при PID.

## Алгоритм

### CO2 Trend Calculator

Каждую 1 минуту замер CO2 попадает в кольцевой буфер (5 значений). Тренд определяется как разница текущего значения и средней 3 старейших замеров:

| Разница | Тренд |
|---------|-------|
| > +15 ppm | `rising` |
| < -15 ppm | `falling` |
| в пределах ±15 | `stable` |

### Зоны CO2 (target по умолчанию 650 ppm)

| Зона | CO2 (при target=650) | Тренд rising | Stable | Falling |
|------|---------------------|-------------|--------|---------|
| Сильное превышение | >800 | +5% | +5% | стоим |
| Умеренное превышение | 700-800 | +2% | стоим | стоим |
| Околоцелевая (deadband) | 600-700 | +2% | стоим | стоим |
| Ниже цели | 500-600 | стоим | стоим | -5% |
| Значительно ниже | <500 | стоим | -5% | -5% |

### Защита от осцилляций

Если за последние 4 оценки направление менялось ≥2 раз (открыть→закрыть→открыть) — привод **не двигается**.

### Rate Limit

- Оценка позиции: каждые 5 минут
- Минимальный интервал между движениями: 4.5 минуты
- Антидребезг встроен в automation condition

## Безопасность

| Событие | Действие |
|---------|---------|
| T улицы < min порога | Закрыть окно + уведомление |
| Перегрузка привода | Закрыть окно + уведомление |
| CO2 сенсор unavailable (>2 мин) | Закрыть окно + уведомление |

### Координация с радиатором

- Окно открыто > 5% → `switch.radiator_left_zap_open_window` = ON (радиатор снижает температуру)
- Окно закрыто < 3% → OFF

## Состав пакета

### Template Sensors

| Sensor | Описание |
|--------|----------|
| `sensor.co2_trend` | Тренд CO2: rising / falling / stable |
| `sensor.ventilation_recommended_position` | Рекомендуемая позиция окна (0-100%) |
| `sensor.ventilation_co2_error` | Отклонение CO2 от цели (ppm) |

| Entity | Описание | Default |
|--------|----------|---------|
| `input_number.ventilation_target_co2` | Целевой CO2 | 650 ppm |
| `input_number.ventilation_min_position` | Минимальная позиция | 0% |
| `input_number.ventilation_max_position` | Максимальная позиция | 80% |

## Установка

1. Скопируйте `packages/ventilation/` в `/config/packages/`.
2. **Перезагрузите Home Assistant** (полный рестарт — reload не создаст input_number и trigger-based sensors).
3. Проверьте, что `sensor.sensor_zap_co2` и `cover.drivent_zap_left_driventokonnyi_privod` доступны.
4. Настройте параметры через UI:
   - Целевой CO2 (по умолчанию 650 ppm)
   - Минимальная уличная температура (по умолчанию -10°C)
   - Максимальная позиция окна (по умолчанию 80%)
